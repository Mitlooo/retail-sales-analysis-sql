# Retail SQL Analysis Project - Explanation

## Project Overview

This project uses MySQL to analyse retail sales order data. The goal is to practise SQL queries and answer business questions about sales, profit, product performance, regional performance, order status, and payment methods.

## Skills Practised

- Creating a database
- Creating a table
- Inserting data
- Selecting columns
- Filtering rows with WHERE
- Aggregating data with COUNT and SUM
- Grouping data with GROUP BY
- Sorting results with ORDER BY
- Filtering grouped results with HAVING

## Business Questions Answered

- What are total sales and total profit?
- What is the overall profit margin?
- Which region has the highest sales?
- Which category has the highest profit?
- Which product has the highest sales?
- Which product has the highest profit?
- Which city has the most orders?
- How many orders were delivered, pending, or returned?
- Which payment method is used most often?
- Which month had the highest sales?

## Simple Interview Explanation

I created a retail sales database in MySQL and wrote SQL queries to analyse sales orders. I used SELECT, WHERE, GROUP BY, ORDER BY, SUM, COUNT, and HAVING to answer business questions about total sales, profit, regions, categories, products, monthly trends, delivery status, and payment methods. This helped me understand how SQL can be used to summarise and analyse business data.
